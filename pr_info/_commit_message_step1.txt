Add mergeable_state field to PullRequestData

Add Optional[str] `mergeable_state` field to the `PullRequestData`
TypedDict and populate it from `pr.mergeable_state` in all five PR
builder methods on `PullRequestManager`:

- create_pull_request
- get_pull_request
- list_pull_requests
- find_pull_request_by_head
- close_pull_request

This surfaces GitHub's PR mergeable_state ("clean", "dirty",
"unstable", "blocked", etc.) on returned PR data so callers can
distinguish a "blocked" state (e.g. unresolved reviews) from the
plain `mergeable` boolean. Foundation for surfacing PR review
feedback in check_branch_status (issue #173).

Tests:
- Extend `create_mock_pr` with default `mergeable_state="clean"`.
- Assert the new field on the existing create/get/list/close/find
  success tests.
- Add a parameterised test verifying values "clean", "dirty",
  "unstable", "blocked", and None flow through unchanged.
