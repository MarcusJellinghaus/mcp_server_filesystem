fix(edit_file): normalize backslashes in file_path on POSIX

On Linux, backslashes are valid filename characters, so passing a
Windows-style relative path like "sub\dir\test_file.py" caused
pathlib.Path to treat it as a single component, leading to a
FileNotFoundError. The diff output normalization was already in
place inside _create_diff, but the path used for the existence
check and read was not normalized.

Replace "\" with "/" at the top of edit_file before path resolution
so the same call resolves to "<project_dir>/sub/dir/test_file.py"
on both Windows and POSIX systems.
