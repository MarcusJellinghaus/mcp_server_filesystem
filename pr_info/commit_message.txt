fix: include hostname in cache_safe_name to prevent cache collisions

Update RepoIdentifier.cache_safe_name to prepend hostname (dots replaced
with underscores), changing format from "owner_repo" to
"hostname_owner_repo". This prevents cache file collisions between repos
with the same owner/name on different GitHub hosts (e.g., github.com vs
ghe.corp.com).

Update TestCacheFilePath assertions and test_cache_safe_name_property to
expect the new format. Add test_cache_safe_name_with_ghe_hostname test.
