feat(read_file): add with_line_numbers formatting with smart defaults

Implement the with_line_numbers parameter for read_file:
- Smart default: True when a line range is given, False for full reads
- Dynamic-width right-aligned line numbers with arrow separator (N→content)
- Width computed from the largest line number in the returned slice

Update existing slicing tests to explicitly pass with_line_numbers=False
to preserve their raw-content assertions. Add 7 new tests covering:
- Default on/off behavior for range vs full reads
- Explicit True/False overrides
- Dynamic width for narrow (1-digit) and wide (3-digit) line numbers
- Empty result when start_line is past EOF with line numbers enabled
