fix(ci): use current branch's mcp_workspace in file-size check

The file-size CI job runs mcp_coder via uvx in isolation, but
mcp_coder depends on mcp_workspace as a transitive dependency.
When uvx resolves mcp_workspace independently, it may install a
version that lacks symbols added in the current branch (e.g.
get_repository_identifier), causing an ImportError.

Add --with . to the uvx invocation so it injects the current
branch's mcp_workspace into the isolated environment, ensuring
compatibility between mcp_coder and the code being tested.
