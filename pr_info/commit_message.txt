feat(read_file): replace f.read() with line-by-line streaming for range slicing

Replace the bulk f.read() call with enumerate(file, start=1) streaming.
When start_line/end_line are provided, only the requested line range is
collected and returned. Breaks early after end_line for efficiency.

- Full reads (no range params) return identical content to before
- Sliced reads clamp end_line to file length, return "" if start > EOF
- Files without trailing newlines are preserved correctly
- Updated logging: sliced reads log the line range instead of byte count

Add 8 unit tests for slicing behavior and edge cases.
